# AI聊天电商项目需求文档（草案）

## 1. 项目目标
- 通过聊天式交互完成商品浏览、检索、加购与下单
- 引入AI服务（DeepSeek优先）与RAG实现语义搜索
- 具备后台监控与心跳注册能力
- 具备基础支付流程（支付宝沙箱）
- 具备商品数据采集与入库能力

## 2. 范围与不在范围
- 范围内：电商业务前端、AI后端、监控服务端与前端、数据采集程序、数据库与向量库
- 不在范围：线上生产部署、复杂的推荐算法、营销活动系统

## 3. 角色与使用场景
- 用户：注册/登录、聊天查询商品、查看购物车、发起支付
- 运营/管理员：查看监控仪表盘、关注服务状态与心跳
- 系统：采集商品数据并入库，向量化并用于语义搜索

## 4. 系统模块
### 4.1 电商业务前端（Vue3）
- 注册/登录
- 商品浏览（列表/详情）
- 购物车管理
- 聊天对话框：通过自然语言完成商品查找、筛选、加购
- 支付：对接支付宝沙箱

### 4.2 AI后端（Spring Boot + WebFlux + LangChain4j）
- 用户与商品业务API
- 聊天服务：AI Service + Skills + MCP调用
- RAG：向量化存储与语义检索
- 支付回调与订单状态更新

### 4.3 监控服务端（Netty）
- 服务器上线注册
- 心跳维持与离线检测
- 状态变更事件

### 4.4 监控前端（Vue3）
- 服务列表与状态概览
- 心跳与告警展示
- 指标仪表盘

### 4.5 数据采集程序（Python爬虫）
- 采集商品基础信息
- 入库并触发向量化流程

## 5. 关键功能需求
### 5.1 账号与权限
- 注册、登录、退出
- 简单角色区分（用户/管理员）

### 5.2 商品与购物车
- 商品列表/详情
- 购物车增删改查
- 订单创建

### 5.3 聊天式电商
- 自然语言查询商品
- 通过聊天执行加购/移除/查看购物车
- 支持商品推荐与相似商品查询（基于向量检索）

### 5.4 支付
- 支付宝沙箱下单
- 支付结果回调与订单状态变更

### 5.5 监控
- 服务注册与心跳
- 服务状态展示
- 异常/离线提示

## 6. 数据与存储
- 关系型数据库：PostgreSQL
- 向量数据库：待定（后续选型）
- 主要数据表：用户、商品、购物车、订单、聊天记录、服务注册/心跳

## 7. 技术选型
- 前端：Vue3 + 脚手架（Vite）
- 后端：Spring Boot + Spring WebFlux + LangChain4j
- 监控：Netty 服务端 + Vue3 仪表盘
- 爬虫：Python
- 数据库：PostgreSQL
- AI：DeepSeek

## 8. 开发与工具
- 前端：VS Code + Trae（豆包）+ Vue 插件
- 后端：IntelliJ IDEA + Trae
- 多项目管理：VS Code Workspace 管理多个 Vue3 项目
- 浏览器：Chrome + "沉浸式翻译"插件

## 9. 非功能需求
- 可扩展：支持新增AI模型与向量库
- 可观测：监控端可查看服务状态
- 安全性：基础鉴权、支付回调校验

## 10. 里程碑建议
- M1：数据库建模 + 基础商品/购物车API
- M2：聊天式检索与加购
- M3：支付与订单闭环
- M4：监控服务上线与仪表盘
- M5：数据采集与向量化流程
